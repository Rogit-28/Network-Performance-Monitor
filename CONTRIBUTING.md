# Contributing

We're excited that you're interested in contributing to this project! All contributions are welcome, from bug reports to feature requests and code changes.

## How to Contribute

To ensure a smooth and collaborative process, please follow these steps:

1.  **Fork the Repository:** Start by creating your own copy of the repository.
2.  **Create a New Branch:** Make a new branch for your changes. A descriptive name is appreciated (e.g., `feature/add-new-chart`, `fix/typo-in-readme`).
3.  **Make Your Changes:** Write your code, fix the bug, or improve the documentation.
4.  **Submit a Pull Request:** When you're ready, open a pull request from your branch to the `main` branch of this repository.

## Style and Conventions

-   **Code:** Please follow the existing code style. We aim for clean, readable, and well-commented code.
-   **Commit Messages:** Write clear and concise commit messages.

## Code of Conduct

This project and everyone participating in it is governed by our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.
